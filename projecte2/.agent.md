## 📋 Responsabilidades

### 1️⃣ Entidad Product
- Definir atributos correctamente con anotaciones JPA
- Incluir ENUM para `condition` (nou, bon estat, acceptable, mal estat)
- Borrado lógico (status boolean)
- Auditoría (dataCreated, dataUpdated)

### 2️⃣ CRUD Endpoints
- `GET /api/products` → todos
- `GET /api/products/{id}` → por ID
- `POST /api/products` → crear
- `PUT /api/products/{id}` → actualizar completo
- `PATCH /api/products/{id}/stock` → actualizar stock
- `PATCH /api/products/{id}/price` → actualizar precio
- `DELETE /api/products/{id}` → borrado físico
- `PATCH /api/products/{id}/logical-delete` → borrado lógico

### 3️⃣ Carga Masiva CSV
- Endpoint `POST /api/products/upload` con multipart file
- Procesar CSV con transacción (@Transactional)
- Rollback si falla cualquier registro
- Retornar: número de registros + errores

### 4️⃣ Query Methods (Query Derivation)
- `findByNameContainsAndStatusTrue()`
- `findByCondition()`
- Ordenamiento por precio/rating

### 5️⃣ JPQL Queries
- Rango de precios (min-max)
- Top productos calidad-precio
- Filtro por rating
- Top mejores valorados

### 6️⃣ Paginación
- Usar `Pageable` y `Page<ProductDTO>`
- Retornar bloques de 5 productos

### 7️⃣ DTOs Obligatorios
- `ProductRequestDTO` (entrada)
- `ProductResponseDTO` (salida)
- `ErrorDTO` (errores)

## ⚙️ Principios de Implementación

✅ **HACER**
- Código simple y legible
- Comentarios en cada clase/método importante
- Separación clara de capas (controller/service/repository)
- Validaciones en DTO con `@Valid` y anotaciones
- ResponseEntity con códigos HTTP correctos
- Transacciones explícitas en operaciones CSV
- Usar Optional correctamente en queries

❌ **NO HACER**
- No añadir funcionalidades fuera de la práctica
- No usar herramientas/patrones no explicados en clase
- No complicar el diseño
- No exponer entidades directamente (usar DTOs)
- No ignorar errores de validación

## 🚨 Errores Típicos a Evitar

| Error | Solución |
|-------|----------|
| `LazyInitializationException` | Usar EAGER o servicios de inicialización |
| CSV sin transacción | Envolver en `@Transactional` |
| DTO expone entity | Usar mapper con DTOs explícitos |
| ResponseEntity sin genérico | Usar `ResponseEntity<DTO>` |
| Queries sin paginación lentas | Implementar Pageable |
| FK sin validar | Usar `@Valid` y excepciones |
